# 游戏机制

*该页面展示的黑魔法均为在游戏内插入后的默认参数*

## 宣告拆字规则

如果是运行时创建的物件，它们都会被统一加进new_splited_event这个组里<br>
你可以操作new_splited_event删除这些没有名字的物件

**虽然这里\"from\"参数是数组，但实际只会拿取第一个元素**

(你当然也可以直接写成字符串，但不知道为什么这样就不能用$指定现有物件了)

```
@[append_split_rule] {
    "from": ["原料物件"],
    "to": ["产出物件1", "产出物件2"]
}
```
注：指定字面：`"字面"`，指定名字：`"$名字"`

## 宜告组字规则

如果是运行时创建的物件，它们都会被统一加进new_merged_event这个组里<br>
你可以操作new_merged_event删除这些没有名字的物件

**虽然这里\"to\"参数是数组，但实际只会拿取第一个元素**

(你当然也可以直接写成字符串，但不知道为什么这样就不能用$指定现有物件了)

```
@[append_merge_rule] {
    "from": ["原料物件1", "原料物件2"],
    "to": ["产出物件"]
}
```
注：指定字面：`"字面"`，指定名字：`"$名字"`

## 宣告成句规则

```
@[append_sentence_rule] {
  
    "text": "成句",
    "has_hint": false,
    "has_animation": true,
    "memory": [],
    "except": "",
    "progress": 0,
    "level": 5
  
}
```

### 宣告成句规则的隐藏机制

"text"字段可以写全角大写的星号“＊”来告诉游戏这个地方如果有任何东西都成立，包括空位。<br>
"except"字段可以限制成句忽略哪些字符（这些字符不能触发成句）<br>
以下示例禁止空位、我、门触发成句，接下来游戏将不会让“＊”算入这些字。<br>
如果有"except"但没有“＊”，成句会创建失败，所以是必须有“＊”才能使用"except"的。

```
"except": "＿我门"
```


"memory"参数可以获取某个字符位置的物件名称至变量。<br>
参考结构："memory": [["变量_1", 2],["变量_2", 10]]<br>
如果该字符位置啥都没有，那么就不会记录变量（保持原样）

```
"memory": [
  ["变量_1", 2],
  ["变量_2", 10]
]
```

实际行为是：<br>
当成句发生时<br>
将变量_1赋值为第3个字符位置的物件名称（索引从0开始）<br>
将变量_2赋值为第11个字符位置的物件名称（索引从0开始）

## 取消成句规则
```
@[delete_sentence_rule] {
  
    "text": "成句"
  
}
```
